品項工具

準備好 Excel / Numbers 檔案，依步驟操作即可一次匯入多筆品項。同名品項自動跳過。

1下載 Excel / Numbers 範本

下載已填好欄位標題及示範資料的 CSV 檔，用 Excel 或 Numbers 開啟後直接填寫，完成後刪除示範列。

欄位 必填 說明
品項名稱必填品項名稱，同名會自動跳過
分類必填必須是系統已存在的分類名稱
顯示單位必填例如：包、瓶、kg
基準單位選填不填則同顯示單位
換算數量選填1 顯示單位 = ? 基準單位，不填預設 1
庫存類型選填冷凍 / 冷藏 / 常溫，不填預設冷凍
安全庫存選填低於此數量顯示警示，不填預設 0
備註選填品項備註說明
越南語名稱選填可留空，之後再用批次翻譯補上
2複製 Gemini 提示詞（將表格轉成 JSON）

在 Excel / Numbers 中全選表格（含標題列）→ 複製 → 打開 Gemini → 貼上以下提示詞，再於最後貼上複製的表格內容，送出後取得 JSON。

3貼入 JSON 並匯入

將 Gemini 回傳的 JSON 貼到下方，按「預覽」確認無誤後再匯入。

系統維護

📋 明細日誌索引（首次設定）

首次使用「員工頁 → 品項明細」功能時，需在 Firebase 建立一次索引，之後永久有效。步驟如下：

  1. 用 Chrome 打開員工頁
  2. 按 F12（Mac 是 Command + Option + J），切到 Console 分頁
  3. 點任意品項的「明細」按鈕
  4. Console 出現紅色錯誤，裡面有一個 console.firebase.google.com/... 連結，按住 Ctrl（Mac 是 Command）點它
  5. Firebase 網頁打開後按「建立索引」
  6. 等 2–3 分鐘，狀態變成 ✅ 就完成了

⚠️ 如果沒有紅色錯誤出現，代表索引已經建好，不需要再做任何動作。
