## 1. 系統簡介

本系統分為兩個頁面：

| 頁面 | 網址結尾 | 使用對象 |
|------|---------|---------|
| **員工頁** | `/index.html`（主頁） | 日常操作員工 |
| **管理後台** | `/admin2.html` | 管理人員 |

**主要功能**：
- 即時查看各倉位庫存
- 記錄入庫、消耗、備料、回放
- 定期盤點
- 查看品項操作明細（7 天內）
- 匯出庫存報表

> 系統資料儲存在雲端（Firebase），所有裝置即時同步。

## 2. 安裝到手機桌面（建議）

安裝後可像 App 一樣直接從桌面開啟，不需要每次打網址。

### iPhone / iPad（Safari）
1. 用 **Safari** 打開員工頁網址
2. 點下方工具列的 **分享按鈕**（方塊加箭頭圖示）
3. 選「**加入主畫面**」
4. 名稱保持預設，點「**新增**」

### Android（Chrome）
1. 用 **Chrome** 打開員工頁網址
2. 點右上角 **三個點**
3. 選「**新增到主畫面**」或「**安裝應用程式**」

> 管理後台也可以同樣方式安裝，建議只安裝在管理人員的裝置上。

## 3. 員工頁操作說明

### 3-1. 頁面總覽

進入後可看到三個頁籤：

- **庫存**：主要操作頁面，顯示所有品項庫存
- **動態**：最近操作記錄
- **盤點**：定期盤點功能（需要 PIN 碼）

右上角功能：
- 🌐 語言切換（中文／越南語）
- **管理** 按鈕：前往管理後台

---

### 3-2. 庫存頁面

顯示所有**啟用中**的品項，依分類排列。

**篩選與搜尋**：
- 上方分類按鈕可篩選特定分類
- 搜尋框可輸入品項名稱快速找到品項

**庫存顯示說明**：
- 每個品項會顯示各倉位（大倉冷凍、大倉冷藏等）的存量
- 存量低於安全線會**標紅色警示**

**倉位說明**：

| 倉位名稱 | 說明 |
|---------|------|
| 大倉冷凍 | 主要冷凍儲存區 |
| 大倉冷藏 | 主要冷藏儲存區 |
| 大倉常溫 | 常溫儲存區 |
| 解凍冷藏 | 備料解凍中的區域 |
| 其他備貨 | 其他已備料存放區 |

---

### 3-3. 操作功能

點擊品項卡片可展開操作區，有以下四種動作：

#### 入庫
- 新到貨品項入庫時使用
- 填入**入庫數量**，選擇單位後按「✅ 確認入庫」
- 庫存會自動加到對應的大倉倉位

#### 備料
- 將品項從大倉移到備料區（解凍或其他備貨）
- 填入**備料數量**，選擇單位後按「🍕 確認備料」
- 需要品項在後台開啟「允許備料」才會出現此按鈕

#### 消耗
- 實際使用品項時記錄消耗量
- 填入**消耗數量**，選擇單位後按「📤 確認消耗」
- 庫存會從對應消耗倉位扣除

#### 回放
- 備料後沒用完，退回大倉時使用
- 填入**回放數量**，選擇單位後按「↩️ 確認回放」
- 需要品項在後台開啟「允許回放」才會出現此按鈕

> **注意**：回放僅限**完整包裝、未開封**且符合保存條件的品項，已開封或分裝的剩餘量不可回放。

---

### 3-4. 快速操作列（桌機版）

桌機寬螢幕下，庫存以表格呈現，每行右側有快速操作欄位，可直接輸入數量後按對應按鈕，不需要展開品項。

---

### 3-5. 品項明細

點擊品項名稱旁的「**明細**」按鈕，可查看該品項**近 7 天**的操作記錄，包含：
- 操作時間
- 操作類型（入庫／消耗／備料／回放）
- 數量
- 備註

> 首次使用明細功能需建立 Firestore 索引，步驟請見「管理後台 → 系統設定 → 系統維護」的說明。

---

### 3-6. 動態頁

顯示全系統最近 80 筆操作記錄，可查看所有品項的最新動態。

---

### 3-7. 語言切換

右上角地球圖示可切換中文／越南語，切換後所有品項名稱、按鈕文字會同步改變（需後台設定越南語翻譯）。

---

## 4. 盤點功能

### 4-1. 進入盤點

1. 點上方「**盤點**」頁籤
2. 輸入**盤點 PIN 碼**（向管理人員取得）
3. 進入盤點頁面

### 4-2. 盤點方式

**盤點頁面**會列出所有啟用中的品項，每個品項顯示各倉位的**目前系統庫存**。

填入實際清點數量後：
- 若數量**與系統相符** → 直接按「**✅ 確認**」
- 若數量**有差異** → 填入實際數量後按「**✅ 確認**」，系統會自動調整庫存並記錄差異

### 4-3. 盤點說明

- 每個品項獨立確認，可以分批完成
- 已確認的品項會顯示綠色勾勾
- 確認後的調整會立即同步到庫存

## 5. 管理後台操作說明

**網址**：主頁網址後加 `admin2.html`

登入需要輸入 **4 位數管理 PIN 碼**。

---

### 5-1. 品項管理

顯示所有品項（含停用）的完整列表。

#### 新增品項
點右上角「**＋ 新增品項**」，填寫以下欄位：

| 欄位 | 說明 |
|------|------|
| 品項名稱 | 必填，顯示在員工頁的名稱 |
| 分類 | 必填，需先在「分類管理」建立 |
| 越南語名稱 | 選填，員工切換越南語後顯示 |
| 儲存類型 | 冷凍／冷藏／常溫，會自動帶入預設倉位 |
| 顯示單位 | 員工看到的單位，例如：包、桶、顆 |
| 基準單位 | 系統內部計算單位（與顯示單位不同時填，例如 g） |
| 換算 | 幾個基準單位 = 1 顯示單位（例如 800g = 1包，填 800） |
| 安全線 | 庫存低於此數量時顯示紅色警示 |
| 允許備料 | 開啟後員工可執行備料動作 |
| 允許回放 | 開啟後員工可將備料退回大倉 |
| 排序 | 數字越小越靠前 |
| 狀態 | 停用的品項不會在員工頁顯示 |

#### 編輯品項
點品項右側「**編輯**」按鈕，修改後按「儲存品項」。

> **注意**：修改倉位設定時，若該倉位仍有庫存，系統會跳出警告確認。

#### 停用 / 啟用品項
點「**停用**」或「**啟用**」按鈕，停用的品項在員工頁隱藏，但資料保留。

#### 刪除品項
點「**刪除**」按鈕，此操作**無法復原**，會永久刪除品項資料。建議改用「停用」代替刪除。

---

### 5-2. 分類管理

管理品項的分類標籤。

- **新增分類**：在下方輸入框填入分類名稱，按「新增」
- **刪除分類**：點分類右側「✕」（僅限沒有品項使用的分類可刪除）
- **分類翻譯**：設定各分類的越南語名稱，提供越南語員工使用

---

### 5-3. 報表

查詢指定時間範圍的庫存進出統計。

**選擇時間範圍**：
- 今天 / 昨天 / 本週 / 本月 / 自訂區間

**表格欄位說明**：

| 欄位 | 說明 |
|------|------|
| 期初庫存 | 查詢期間開始時的庫存（由現在庫存反推） |
| 入庫 | 查詢期間的總入庫量 |
| 消耗 | 查詢期間的總消耗量 |
| 回放 | 查詢期間的總回放量 |
| 期末庫存 | 目前實際庫存 |

點欄位標題可排序，點「**匯出 Excel**」可下載報表。

> **注意**：備料動作不計入報表（只是倉位之間移動，不是消耗）。

---

### 5-4. 系統設定

#### PIN 碼管理
- **管理後台 PIN**：登入後台使用，預設 `1234`，請務必修改
- **盤點 PIN**：員工進入盤點頁面時使用，可獨立設定或清除

修改方式：
1. 填入新 PIN 碼（4 位數字）
2. 再次輸入確認
3. 按「儲存」

#### 品項工具

**匯出備份**：將所有品項資料匯出為 JSON 檔案，建議定期備份。

**批次匯入品項**（適合初次大量建立品項）：
1. 下載 CSV 範本
2. 在 Excel 填入品項資料
3. 複製提示詞，連同表格內容一起貼給 Gemini AI
4. 將 Gemini 回傳的 JSON 貼入匯入框
5. 按「預覽」確認後按「確認匯入」

#### 越南語翻譯
為品項和分類新增越南語名稱。

1. 按「① 產生 Gemini 提示詞」
2. 複製提示詞，貼給 Gemini
3. 將 Gemini 回傳的翻譯結果按「② 匯入翻譯結果」貼入

#### 系統維護

**明細日誌索引（首次設定）**：
首次使用員工頁品項明細功能時，需在 Firebase 建立一次索引：
1. 用 Chrome 打開員工頁
2. 按 F12（Mac 是 Command + Option + J），切到 Console 分頁
3. 點任意品項的「明細」按鈕
4. Console 出現紅色錯誤，裡面有 Firebase 連結，按住 Ctrl（Mac 是 Command）點它
5. Firebase 網頁打開後按「建立索引」
6. 等 2–3 分鐘，狀態變成 ✅ 就完成了

> 如果沒有紅色錯誤出現，代表索引已建好，不需要再做。

**操作記錄管理**：
- 「查詢記錄筆數」：查看系統目前有多少操作記錄
- 「清理 6 個月前記錄」：刪除舊記錄釋放空間（**無法復原**）

## 6. 常見問題

### Q：畫面頂端出現紅色「目前離線」橫幅
**A**：手機或平板網路斷線，操作無法儲存。確認 WiFi 或行動數據後，橫幅會自動消失。

---

### Q：點操作按鈕沒有反應
**A**：確認是否在離線狀態（頂部有紅色橫幅）。若有網路仍無反應，重新整理頁面。

---

### Q：品項明細點了沒有資料或出現錯誤
**A**：可能需要建立 Firestore 索引，請參考「管理後台 → 系統設定 → 系統維護」的步驟。

---

### Q：員工頁看不到某個品項
**A**：該品項可能被停用，請到「管理後台 → 品項管理」確認狀態。

---

### Q：庫存數字感覺不對
**A**：可以到「管理後台 → 報表」查看期間進出記錄，確認哪個時段有異常。若確認需要修正，可透過「盤點」功能將庫存調整為實際數量。

---

### Q：忘記 PIN 碼
**A**：請聯絡系統管理員重設。若完全無法登入後台，需要請工程師直接到 Firebase 後台查看設定值。

---

### Q：如何讓新員工使用系統
**A**：
1. 把員工頁網址傳給員工
2. 請他用 Safari（iPhone）或 Chrome（Android）開啟，並加入主畫面
3. 需要使用盤點功能時，提供盤點 PIN 碼

---
